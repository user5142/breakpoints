Fixing Fotomatic

Ben Hatch

Overview

This article provides an approach and step-by-step solution to Fotomatic, an off-platform project that I completed as part of Codecademy’s Full Stack Software Engineer Career Path.

The task for this project was to fix a broken version of a responsive website for a fictitious camera company called Fotomatic.

Codecademy provided a download to the “broken code”, which included an index.html file, two CSS stylesheets, and an images folder. They also provided a spec sheet to follow in making adjustments to the broken code, as well as a link to a working version of the final product.

I decided to write up a "work instructions" style tutorial for Fotomatic as I completed the project for two main reasons:

1) I noticed on the forums that many of my coursemates had struggled with this project, so I figured a step-by-step solution would be helpful. The final solution code provided by Codecademy also seemed to be missing several key spec sheet requirements, adding some confusion and frustration to the mix.

2) Forcing myself to devise and write up a web development process seemed like a great learning opportunity.

Execution

Here is the step-by-step approach I took to debug and develop Fotomatic per the spec sheet guidelines.

Part I: Debug

1) The style.css file is not linked to index.html. Add the following code to the <head> element:

<link rel="stylesheet" href="./resources/css/style.css">

2) The viewport meta tag is missing from index.html. Add the following code to the <head> element:

<meta name="viewport" content="width=device-width, initial-scale=1, minimum-scale=1">

3) The small Instagram icon at the end of the nav bar is misaligned with the nav links. Add the following declaration to the header .icon ruleset within the /* Header */ section of style.css:

vertical-align: middle;

4) The Features Section images are not linked properly in index.html. Correct the Features Section <img> elements:

<img src="./resources/images/feature-1.png"/>
            <img src="./resources/images/feature-2.png"/>

5) The Quotes Section comment is missing a closing tag. Correct the comment:

<!-- Quotes Section -->

6) The media query within the /* Header */ section of style.css is intended for the mobile version, but has the incorrect max-width. Update the media query max-width from 760px to 450px.

7) The first media query within the /* Features */ section of style.css is intended for mobile, but has the incorrect max-width. Update the max-width from 890px to 450px.

8) Three class selectors are written incorrectly within the /* Quotes Section */ of style.css. Correct them to the following (each selector appears twice within the section:

#quotes-section .content
            #quotes-section .quote
            #quotes-section .quote-citation

Part II: Develop

Landing Page

Given the pre-assigned media queries and spec sheet guidelines, I defined the landing page (desktop) viewport for this project as screen sizes greater than 760px in width.

/* Header */

1) The <header> element is slightly too large (86px) and needs to be adjusted per the spec sheet to 85px. In the header .content ruleset, remove the padding declaration and replace it with the following:

height: 5.3125rem;

2) Now we need to add padding back in to properly space the navigation bar per the spec sheet. Add the following padding declaration to the header .content ruleset:

padding: 0 1.875rem;

3) The Fotomatic logo needs some updates.

First, navigate to Google Fonts and search for Roboto Mono. Click into the font and scroll down until you find the Regular 400 style. Click the Select Regular 400 link, then follow the instructions on the pop-out window to add the code to index.html and style.css. The CSS code should be added to the header .logo ruleset within the /* Header */ section.

Next, add the following code to the header .logo ruleset to update the font-size and color of the logo, and to enhance its alignment within the nav bar:

font-size: 1.5rem;
            color: #4a4a4a;
            padding-bottom: .2rem;

4) Although the spec sheet doesn’t call for it, the provided working version has a fixed navigation bar, so I opted for the same. Add the following declaration to the header ruleset:

z-index: 100;

/* Sign Up Section */

1) To populate the background banner image, add the following declarations to the #sign-up-section ruleset:

background-image: url(../images/banner-landingpage.jpg);
            background-position: center;
            background-repeat: no-repeat;
            background-size: cover;

2) To reposition the CTA, add the following declarations to the #sign-up-cta ruleset:

margin-top: 10.625rem;
            margin-left: 6.25rem;

/* Features Section */

1) To correct the overflow issue of this section, navigate to /* Universal Styles */, then add the following declaration to the .image-container img ruleset:

max-width: 100%;

2) The bottom padding of the Features Section container needs to be updated. Update the padding declaration within the #features-section ruleset to the following:

padding: 4rem 5% 2.875rem;

3) The gap between the two feature image containers needs to be fixed.

First, remove the margin declaration within the .feature ruleset.

Next, add the following declaration to the #features-section ruleset:

gap: 38px;

4) Padding needs to be added to the top and bottom of the text content within the feature containers. Add the following ruleset within the /* Features Section */:

.feature .content {
                padding: 2.5rem 0;
            }

/* Filters Section */

1) To make the images display next to each other on the same row, add the following declaration to the #filters-section .images-container ruleset:

display: flex;

2) To create a bit more spacing between the text elements, add the following declaration to the #filters-section .content h3 ruleset:

padding-top: .75rem;

/* Quotes Section */

1) Cut the padding declaration from the #quotes-section .content rule, then paste it into the #quotes-section ruleset.

#quotes-section {
                background-color: #f3f3f3;
                padding: 5rem 0;
            }

2) Add the following declarations into the #quotes-section .content ruleset:

display: flex;
            justify-content: center;
            padding: 0 1.875rem;

/* Footer */

1) The footer media query is causing the footer text to display incorrectly (too small) for the landing page (desktop) viewport.

Within the footer media query, change min-width to max-width.

Tablet

Given the pre-assigned media queries and spec sheet guidelines, I defined the tablet viewport range for this project as screen sizes between 450px - 760px in width.

/* Header */

1) Add a new media query within the /* Header */ section of style.css for tablet-sized screens:

@media only screen and (max-width: 760px) {
            }

2) Within the tablet media query, add the following rulesets:

To decrease the height and side padding of the navigation bar:

header .content {
                height: 4.5rem;
                padding: 0 1rem;
            }

To decrease the size of the logo:

header .logo {
                font-size: 1.25rem;
            }

To decrease the padding between the navigation links:

nav li {
                padding-left: 1rem;
            }

3) The Join Us button is missing from the tablet navigation bar. Follow these steps to make the Join Us button appear on tablet-sized screens:

In index.html, add the following code as a fourth li element to the ul within <nav class=”desktop”>:

<li class=”tablet”><a href="#" class="button">Join us</a></li>

To hide the Join Us button on large screens, add the following rule to the /* Header */ section within style.css: 

header nav .tablet {
                display: none;
            }

To display the Join Us button on tablet-sized screens, add the following rulesets within the /* Header */ tablet media query:

header .button {
                padding: .1875rem .5rem;
                background-color: #9dc20b;
                line-height: 1.6;
                color: white;
            }
                
            header nav .tablet {
                display: inline;
            }

4) Between around 550px - 450px, the tablet navigation menu experiences some display issues. To hide the logo and center the remaining navigation menu within this range, add the following media query to the /* Header */ section:

@media only screen and (max-width: 540px) {
             .logo {
                display: none;
             }
                
             header .content {
                justify-content: center;
                padding: 0 0;	
             }
            }

/* Sign Up Section */

1) To align the top of the main content with the bottom of the nav bar, navigate to /* Universal Styles */ and find the max-width: 760px tablet media query. Then, update the top declaration to match the tablet adjusted height of the nav bar:

top: 4.5rem;

2) To align the text in the center of the section, add the following declarations to the #sign-up-cta ruleset within the tablet media query:

margin: 0;
            padding: 0;

/* Features Section */

1) To align the feature containers next to each other, delete all declarations from the #features-section ruleset within the tablet media query, then add in the following declarations:

padding: 1rem;
            gap: 1rem;

2) Add the following rulesets within the tablet media query to style the feature image containers per the spec sheet:

.feature h2 {
                font-size: 1.5rem;
                line-height: 1.3;
                font-weight: bold;
            }
                
            .feature h3 {
                font-size: .875rem;
                line-height: 1.4;
            }
                
            .feature .image-container {
                width: 100%;
            }
                
            .feature .content {
                text-align: left;
                padding-top: 1rem;
            }

/* Filters Section */

Make the following updates to the tablet media query within the /* Filters Section */of style.css:

1) Update the padding declaration within the #filters-section ruleset to the following:

padding: 4rem 0 0 0;

2) Update the margin-bottom declaration within the #filters-section .content ruleset to the following:

margin-bottom: 4rem;

3) Update the font-size and font-weight declarations within the #filters-section .content h2 ruleset to the following:

font-size: 2rem;
            font-weight: bold;

4) Add the following declaration to the #filters-section .content h3 ruleset:

padding-top: 0;

5) Comment out the last #filters-section .extra rule. This will be utilized later for the mobile media query.

/* #filters-section .extra {
                display: none;
            } */

/* Quotes Section */

Make the following updates to the tablet media query within the /* Quotes Section */ of style.css:

1) Add the following new rules:

#quotes-section {
                padding: 0;
            }
              
            .content .quote {
                text-align: left;
            }

2) Within the #quotes-section .content rule, update the padding declaration to the following:

padding: 1.875rem 1.75rem;

/* Footer */

1) The media query within the /* Footer */ section is intended for the mobile version, but has the wrong max-width. Update the max-width from 760px to 450px, which will apply the smaller font-size to the mobile version and correct the font-size for the tablet version.

Mobile

Given the pre-assigned media queries and spec sheet guidelines, I defined the mobile viewport for this project as screen sizes smaller than 450px in width.

/* Header */

1) To attain the proper nav bar height per the spec sheet, adjust the header .content ruleset within the mobile (max-width: 450px) media query to the following: 

header .content {
                padding: 0;
                height: 3rem;
            }

2) Add the following rule to the mobile media query to smooth the alignment of the icons with the Join Us button:

nav li a img {
                vertical-align: middle;
            }

/* Sign Up Section */

1) Add the following media query to the /* Universal Styles */ section to attain the proper top offset of the main content section for mobile. 

@media only screen and (max-width: 450px) {
             .main-content {
                top: 3rem;
             }

2) To reposition the background banner image per the spec sheet mockup, add the following rule to the mobile media query within the /* Sign Up Section */:

#sign-up-section {
                background-position: right;
            }

/* Features Section */

1) First, update the tablet media query to include a min-width value of 450px:

@media only screen and (max-width: 760px) and (min-width: 450px) {
            }

2) To stack the feature containers on top of each other, add the following ruleset to the mobile media query within the /* Features Section */:

#features-section {
                flex-flow: column;
                padding: 0;
                gap: 0;
            }

3) To adjust the padding around the feature image containers, add the following rule to the mobile media query:

.feature {
                padding: 1rem 0 1.875rem;
            }

4) To adjust the spacing of the feature image container text, add the following rule to the mobile media query:

.feature .content {
                padding: 1.875rem 2rem 0;
            }

/* Filters Section */

1) To smooth out the text and spacing, add the following mobile media query to the /* Filters Section */:

@media only screen and (max-width: 450px) {
             #filters-section {
                padding: 2rem 0 0 0;
             }
                
             #filters-section .content {
                margin-bottom: 2rem;
                padding: 0 1.5rem;
             }
                
             #filters-section .content h2 {
                font-size: 1.25rem;
             }
            }

2) Cut the #filters-section .extra rule that was commented out during the tablet media query sequence, then paste it into the mobile media query within the /* Filters Section */. Remove the comment tags to make the rule active.

/* Quotes Section */

Add the following mobile media query to properly style and space the elements within the quote section:

@media only screen and (max-width: 450px) {
             #quotes-section .content {
                flex-flow: column;
             }
                
             #quotes-section .quote {
                padding: 0;
             }
                
             .content .quote {
                text-align: center;
             }
            }

Insights

As I have done with a few other Codecademy projects, I initially jumped right into this one, guns blazing, and made some good progress, but then got stuck, spun my wheels for a bit, and ended up starting over from scratch.

I have now realized through repeating this pattern a few times that getting stuck and de-railed is often a result of trying to speed through projects as fast as possible without trying to comprehend the project material.

A better approach, it seems, is to slow down, think things through, and seek to understand each concept as you build and debug. Everything began to click when I committed to going slower and steadier on my second go-around.

Writing out step-by-step instructions on how to complete Fotomatic as I went through it greatly enhanced my comprehension of the project concepts and material. It also helped me create and follow a clear debugging and development process, which seems to be a force multiplier for conquering tricky projects of this caliber.
